1. Démarrage

Double-cliquez sur SovereignOS.Desktop.exe. L'application démarre un serveur HTTP local sur 127.0.0.1:8787 et ouvre l'interface dans une fenêtre native.

Poste cible Windows 10 ou 11 x64. Le runtime .NET est embarqué, aucune installation préalable n'est requise. WebView2 Runtime est présent par défaut sur Windows 11 et Windows 10 à jour.

2. Workflow complet

  1. Sélection des dossiers ATIH · onglet Modo Files, bouton Sélectionner un dossier. Les dossiers retenus doivent se trouver sous l'espace de travail configuré (par défaut %LOCALAPPDATA%\SovereignOS\workspace).
  2. Scan · l'application énumère récursivement les fichiers .txt et .csv, identifie automatiquement le format ATIH par convention de nommage.
  3. Traitement · chaque fichier est parsé en parallèle, les couples (IPP, DDN) sont extraits et agrégés dans le Master Patient Index (MPI).
  4. Identitovigilance · onglet dédié · chaque IPP associé à plusieurs DDN est listé comme collision. Choisir la DDN pivot (manuellement ou via auto-résolution par fréquence).
  5. Export PMSI Pilot · export CSV normalisé avec la DDN pivot injectée, prêt à être téléversé vers e-PMSI.
  6. Purification .txt · réécriture du fichier d'origine en largeur fixe avec la DDN pivot normalisée.

4. Identitovigilance (MPI)

Le MPI agrège toutes les observations (IPP, DDN, source) issues des fichiers traités. Une collision apparaît lorsqu'un même IPP est associé à plusieurs DDN distinctes.

Résolution

  • Auto-résolution · la DDN la plus fréquemment observée gagne le statut pivot. Rapide, fiable sur gros volumes.
  • Résolution manuelle · l'utilisateur peut forcer une DDN pivot depuis l'onglet Identitovigilance.
  • Pas de résolution · l'export conserve les DDN d'origine, flag chaque ligne comme ambiguë pour revue DIM.
Aucun dédoublonnage sur DxCare/CPage Sovereign OS n'écrit jamais dans la facturation (CPage) ni dans le dossier patient (DxCare). L'outil opère exclusivement sur les exports ATIH et produit des exports CSV/.txt destinés à e-PMSI.

5. Exports

Export CSV (PMSI Pilot)

  • Colonnes · IPP, DDN_PIVOT, DDN_OBSERVATIONS, SOURCE_FILES.
  • Séparateur par défaut · ; (compatible Excel FR).
  • Injection formule Excel bloquée · les valeurs commençant par =, +, -, @, tabulation ou retour chariot sont préfixées par une apostrophe.

Purification .txt

Le fichier d'origine est réécrit ligne par ligne en remplaçant la DDN par la DDN pivot du MPI. Les lignes invalides ou de padding sont conservées telles quelles. Encodage latin-1 préservé.

HTML → PDF (dashboards)

Les dashboards exportés depuis BIQuery (Enregistrer la page sous HTML dans Chrome ou Edge) peuvent être convertis en PDF propre via l'onglet Export PDF. Le moteur de rendu est Chromium (identique à celui qui affiche l'UI), les polices web et les graphiques Chart.js sont préservés.

6. Sécurité

  • Le serveur HTTP local est lié à 127.0.0.1 exclusivement. Un en-tête Host non conforme renvoie un statut 421 (défense contre le DNS rebinding).
  • Un token Bearer est généré aléatoirement à chaque démarrage et injecté dans la fenêtre WebView2. Toute requête sans token valide renvoie 401.
  • Tous les chemins de fichier fournis par le frontend sont validés contre la racine de l'espace de travail. Les chemins UNC et les flux de données alternatifs (ADS) sont rejetés.
  • Aucune donnée patient ne quitte le poste. L'application ne fait aucun appel réseau sortant, sauf à charger les polices Google Fonts et les CDN Tailwind/Chart.js à l'ouverture.
Données sensibles Le MPI contient des couples (IPP, DDN) protégés par l'article 9 du RGPD. Vider le MPI (Reset) à chaque fin de session partagée. Les exports CSV et .txt doivent suivre la procédure DIM locale pour l'archivage et la destruction.
